How I've used XML in this website

The motivating factor for the use of XML in my website is boilerplate formatting. As described my previous blog post, the introduction of RSS and the speculation of post categories made the hard-coded creation of these posts difficult. By creating an XML Schema, I've limited the input content to only the article and meta-data such as the title, description, and publication date.

I call this Schema the simple weblogging language (SWL). It's nothing really special: the root element swl (in namespace https://www.dfargo.com/ns/swl) contains an item element from the RSS namespace and an article node from the XHTML namespace.

Using these two elements, I can apply XSLT stylesheets such as the one for this page using the site's Makefile to generate formatted XHTML output.

Besides the clear advantage of cutting down on maintenance, the use of XML (specifically with XSLT stylesheets) allows me to use XInclude elements, or embedded documents without the need of spotty browser support for XHTML/XLinks.
